Se-Go (Server-in-Go) — A minimalist Go server for turning HTML/JS into system GUIs. A secure bridge between the web and Linux without Electron or dependencies.

# SeGO (Server in GO) 🚀

### Electron жрёт диск, Tauri — мозг. Наш сервер просто работает. Сделай системное GUI за 5 минут.

**SeGO** — это ультралегкий (6MB) мост, превращающий обычный HTML/JS в полноценный пульт управления вашим Linux. Никаких `node_modules`, никакой компиляции фронтенда. Только чистая мощь Go и простота Vanilla JS.

## 🧱 Сердце проекта — Go
В основе **Se-Go** лежит производительный сервер, написанный на **Go**. Мы не используем сторонние библиотеки — только стандартную поставку языка (`net/http`, `os/exec`). Это гарантирует:
* **Стабильность**: Сервер работает как часы, не требуя установки рантаймов.
* **Безопасность**: Весь системный ввод-вывод контролируется строго типизированным кодом Go.
* **Компактность**: Весь сервер компилируется в один бинарник, который весит меньше, чем иконка в Electron-приложении.

## 🛠 Порог входа: НУЛЕВОЙ (Метод AI-разработки)

SeGO спроектирован так, чтобы его понимал любой современный ИИ (ChatGPT, Claude, DeepSeek). Вам не нужно быть программистом, чтобы «оседлать» систему.

1. **Покажите ИИ код сервера**: Скопируйте содержимое `main.go` и отправьте нейронке с фразой:
> *"Это мой сервер SeGO на Go. Напиши для него один файл index.html на Vanilla JS, который будет делать [ВАША ЗАДАЧА]. Используй fetch('/api/run') для команд."*


2. **Получите результат**: ИИ идеально знает Vanilla JS и создаст интерфейс за секунды.
3. **Запустите**:
* Создайте папку `ui` рядом с бинарником `sego`.
* Положите туда `index.html`.
* Запустите лаунчер: `./sego-run.sh`



> **ВАЖНО**: Для идеального "suckless-вида" установите браузер **surf** (`sudo apt install surf`). Скрипт запустит его в режиме нативного приложения. Если его нет — откроется обычный браузер.

## 🔒 Безопасность (Technical & Honest)

SeGO — это мощный инструмент, который снимает ограничения "песочницы" браузера, оставаясь при этом предельно честным с системой:

* **Локальный биндинг**: Сервер слушает только `127.0.0.1`. Извне к вам никто не пролезет.
* **Авто-токенизация**: При каждом старте — новый секретный ключ. Лаунчер сам прокидывает его в браузер. Сторонний сайт не сможет украсть управление (защита от CSRF).
* **Уважение к правам ОС**: SeGO не «читерит». Если ваш пользователь не имеет прав на удаление файла в `/usr/local/bin`, то и через GUI этот файл удалить не получится. Сервер — это прозрачное зеркало ваших системных прав.
* **Сессионные Cookie**: После входа ставится защищенная кука `SameSite=Strict`.

⚠️ **ЗОЛОТОЕ ПРАВИЛО:**
SeGO выполняет команды с теми правами, с которыми запущен. Если вам нужен «Режим Бога» для системных папок — используйте `pkexec` в своих JS-запросах, чтобы система запросила пароль.

## 🎯 Почему SeGO?

* **Вес**: 6MB против сотен мегабайт Electron.
* **Память**: Потребляет в 15-20 раз меньше ОЗУ.
* **Скорость**: Мгновенный запуск. Идеально для Live-систем (MiniOS, AntiX, Puppy).
* **Прозрачность**: Никакой магии. Вы точно знаете, какая команда ушла в систему.
